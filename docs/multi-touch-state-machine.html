3 ACTIVE 超时 → HIJACK 流程

4 Action 事件 × 状态 转移表

Action 事件 INACTIVE WAITING ACTIVE HIJACK BLOCKING
ACTION_DOWN → WAITING （不消费） → WAITING（reset 新序列） → WAITING（reset 新序列） → WAITING（reset 新序列） → WAITING（同 INACTIVE）
ACTION_POINTER_DOWN 不动 （不消费） 达 min 阈值 → ACTIVE （消费）；否则留 WAITING 手指数 > max → replayAll → INACTIVE；超时 → injectCancel → HIJACK；否则留 ACTIVE （消费） 追踪新指；留 HIJACK （消费，不录制） 留 BLOCKING （劫持抛弃）
ACTION_MOVE 不动 （不消费） 同步坐标；留 WAITING （不消费） 超时 → injectCancel → HIJACK；否则留 ACTIVE （消费） 同步坐标；留 HIJACK （消费，不录制） 留 BLOCKING （劫持抛弃）
ACTION_POINTER_UP 不动 （不消费） 该指入 ignoredIds；留 WAITING → finishGesture → BLOCKING/INACTIVE （消费） → finishHijack → BLOCKING （消费） 留 BLOCKING （劫持抛弃）
ACTION_UP → INACTIVE （不消费） → INACTIVE （不消费） → finishGesture → BLOCKING/INACTIVE （消费） → finishHijack → BLOCKING （消费） 留 BLOCKING （劫持抛弃）
ACTION_CANCEL 任意状态 → reset() → INACTIVE（不消费，交系统处理）
