2 WAITING → ACTIVE：注入 ACTION_CANCEL + off-screen 抬起

WAITING 期间所有事件透传给 App，App 已持有进行中的触摸。当 POINTER_DOWN 使手指数达到 min 阈值，立即转入 ACTIVE：先注入 ACTION_CANCEL （携带当前所有按下手指的坐标 + 序列 downTime）取消 App 端触摸，随后注入连贯的多指抬起序列把每根手指抬离屏幕 （POINTER_UP ×N−1 + UP ×1，每指坐标置屏幕右下角之外）；同时取消 waitingTimeout 计时器。

CANCEL 指针集 = 当前仍按下的手指（pointers 中各项的 currentX/Y）。 WAITING 期间已抬起的“忽略”手指此前已收到 POINTER_UP，App 不再追踪，故不纳入。

抬起序列按真实多指抬起顺序构造（始终抬起当前指针集第 0 个，actionIndex=0），指针集逐个递减， 各事件共享序列 downTime、eventTime 自触发事件起 +1ms 递增。off-screen 坐标取 displayMetrics 屏幕右下角之外。

3 ACTIVE 抬手判定流程（finishGesture）

处于 ACTIVE 时收到 POINTER_UP / ACTION_UP 触发判定。 无论识别结果如何，一律进入 BLOCKING。不再重放、不再注入 CANCEL（CANCEL 已在进入 ACTIVE 时注入）。

4 Action 事件 × 状态 转移表

Action 事件 INACTIVE WAITING ACTIVE BLOCKING
ACTION_DOWN → WAITING（arm waitingTimeout） （不消费） → WAITING（reset 新序列） → WAITING（reset 新序列） → WAITING（reset 新序列）
ACTION_POINTER_DOWN 不动 （不消费） 达 min 阈值 → ACTIVE（注入 CANCEL + 取消计时器） （消费）；否则留 WAITING 手指数 > max → 识别失败 → BLOCKING；否则追踪新指，留 ACTIVE （消费） 留 BLOCKING （劫持丢弃）
ACTION_MOVE 不动 （不消费） syncPointers 同步坐标；留 WAITING （不消费） syncPointers 同步坐标；留 ACTIVE （消费，丢弃） 留 BLOCKING （劫持丢弃）
ACTION_POINTER_UP 不动 （不消费） 移除该指；留 WAITING （不消费） → finishGesture → BLOCKING （消费） 留 BLOCKING （劫持丢弃）
ACTION_UP → INACTIVE （不消费） → reset → INACTIVE （不消费） → finishGesture → BLOCKING （消费） → reset → INACTIVE （消费）
ACTION_CANCEL 任意状态 → reset()（cancelWaitingTimeout + 清空所有数据）→ INACTIVE（不消费，交系统处理）

5 线程安全

机制 说明
synchronized(stateLock) 保护所有可变状态（state、pointers、downTime）的读写，确保 hook 线程与主线程（计时器回调）之间的可见性和原子性。
@Volatile private var state getState() 快速路径直接读取 volatile 字段，无需加锁；写入仍通过 synchronized(stateLock)。
waitingTimeout 原子 claim waitingTimeoutRunnable 在 synchronized(stateLock) 内检查并修改 state：仅当 state 仍为 WAITING 时执行清理，否则跳过。避免过时计时器误操作。
cancelWaitingTimeout() reset()、enterBlocking()、enterActive() 均调用此方法取消计时器（幂等）。
